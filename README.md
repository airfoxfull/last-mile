# Last Mile

> AI-native 软件开发流水线 — 最后一百米的标准化落地

## 问题

50 人以上的软件开发团队用 AI 编码，最大的成本不是 token 费，是**返工**。

一个需求经过前端→后端→测试→安全多个环节，任何一环理解偏差，下游全部白做。当前 AI 工具只管"写代码"，不管"写对代码"——Agent 拿到任务就干，不规划、不沟通、不对结果负责，返工率居高不下。

## 解决方案

Last Mile 通过**强制规划、链式验收、及时反馈**，把返工率降下来。

### 链式责任流转
需求从消费端出发，由近到远流转。上游为下游定清目标，下游不用猜。上游验收下游产出，偏差在当环发现，不传递到下游。

### 强制规划
Agent 动手前必须提交执行计划，人类审阅后才能开工。不是"建议你先想想"，是系统级强制——不提计划就不能执行。

### 满意度评分
人的角色 = 满意度评价 + 方向纠偏。每个阶段完成后人类打分，低分立即修正，不等到最后才发现问题。

### 预算激励
Token 预算不是为了省钱，是为了激发规划意识。有预算约束的 Agent 会更谨慎地规划、更精确地执行。高满意度获奖励，低满意度受惩罚——驱动 Agent 对交付质量负责。

### 知识自进化
每次运行的经验自动沉淀为公司级资产。同样的坑不踩两次。

## 目标用户

**50 人以上的软件开发团队**，有多角色协作（前端/后端/测试/安全/运维），需求传递容易走样，返工成本高，管理层需要可视化流转状态。

## 架构

```
层级 1: 工作流引擎（强制协议）
  通过持久执行引擎保证每一步不可跳过
  Agent 不能绕过"读交接→提计划→等审批→执行→写报告"的流程

层级 2: Agent 自治与进化
  Agent 有自我意识（知道自己的预算、技能、约束）
  自进化机制让 Agent 越用越好

层级 3: 编码执行
  对接多种 AI 模型（Claude/Codex/Cursor/Gemini）
  隔离工作区、会话管理、成本追踪

层级 4: Last Mile 业务逻辑（差异化）
  链式流转、预算激励、满意度评分、知识自进化
```
