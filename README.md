# Last Mile

> AI-native 软件开发流水线 — 最后一百米的标准化落地

## 愿景

让任何开发团队都能轻松转型 AI-native。不做大模型，做标准化落地。

当前 AI 编码能力已经很强（前 90%），但"最后一百米"——让普通开发者和公司真正转型为 AI-native——仍然是空白。现状像送外卖送到小区门口就不管了，用户得自己挖掘各种 skill 和工具才能勉强提效。

Last Mile 解决这个问题。

## 核心机制

### 链式责任流转
需求从消费端出发，由近到远流转。每一环为上游负责，上游验收下游。需求传递不走样。

### 预算激励
Token 奖惩驱动 Agent 自主性。高满意度获奖励，低满意度受惩罚。预算决定生存和晋升。Agent 会精打细算、做好规划。

### 满意度评分
人的角色 = 满意度评价 + 方向纠偏。不做执行，只做评价。

### 知识自进化
每次运行的经验自动沉淀为公司级资产。系统越用越好，不重复犯错。

### 工作流强制
通过持久执行引擎（非 Agent 自觉）保证每一步不可跳过。Agent 不能绕过协议直接工作。

## 架构

```
层级 1: 工作流引擎（强制协议）
  DeerFlow/LangGraph 图结构 或 Hatchet DAG

层级 2: Agent 自治与进化（灵魂）
  借鉴 Clawith: MorphAgent 自进化、SOUL/MEMORY 持久化

层级 3: 编码执行（干活）
  Paperclip 适配器: Claude/Codex/Cursor/Gemini

层级 4: Last Mile 业务逻辑（差异化）
  链式流转、预算激励、满意度评分、知识自进化
```
